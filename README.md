# Baby Name Gender Prediction & Generation

A comprehensive deep learning project exploring various RNN architectures for predicting gender from baby names and generating new names.

## 🎯 Overview

This project explores the application of various Recurrent Neural Network (RNN) architectures for both **gender classification** and **name generation** tasks. It implements and compares Vanilla RNN, GRU, and LSTM models, analyzing their performance in understanding linguistic patterns in names.

### Key Research Questions:
1. Which RNN architecture performs best for gender classification from names?
2. How do different RNN models handle the vanishing gradient problem?
3. What is the quality of names generated by each model?
4. Does multi-task learning improve model performance?
5. How do different loss functions affect gender classification?

## ✨ Features

### Model Implementations
- **Vanilla RNN** - Custom implementation from scratch
- **GRU (Gated Recurrent Unit)** - Built with PyTorch
- **LSTM (Long Short-Term Memory)** - Built with PyTorch
- **Multi-task LSTM** - Combined gender classification and next-character prediction

### Name Generation
- Next-character prediction
- Temperature-controlled sampling for diverse name generation
- Multiple models for comparison

### Model Evaluation
- **Name Quality Score (NQS)** - Custom metric evaluating:
  - Fluidity (name-like appearance)
  - Plausibility (real-name likelihood)
  - Diversity (variety of generated names)
  - Gender Consistency (prediction alignment)

- **Perplexity** - Model surprise metric
- **AUC-ROC** - Classification performance
- **Gradient Analysis** - Vanishing gradient investigation

## 📊 Dataset

**Source**: US Baby Names Dataset (clean version)
- **Total names**: 6,782
- **Gender distribution**: 
  - Girls: 49.3%
  - Boys: 50.7%
- **Name length**: 2-11 characters
- **Preprocessing**:
  - Lowercase conversion
  - Start/end tokens (`!` and `#`)
  - Padding to maximum length
  - Character-level tokenization (29 unique tokens)

## 🧠 Models Implemented

### 1. Vanilla RNN (Custom Implementation)
- **Architecture**: Custom RNN with manual backpropagation
- **Hidden dimension**: 128
- **Purpose**: Baseline model for comparison
- **Performance**: AUC-ROC 0.52

### 2. GRU Model
- **Architecture**: PyTorch GRU implementation
- **Hidden dimension**: 128
- **Embedding dimension**: 64
- **Dropout**: 0.2
- **Performance**: AUC-ROC 0.80

### 3. LSTM Model
- **Architecture**: PyTorch LSTM implementation
- **Hidden dimension**: 128
- **Embedding dimension**: 64
- **Dropout**: 0.2
- **Performance**: AUC-ROC 0.84

### 4. Multi-task LSTM
- **Dual objectives**: Gender classification + Next-character prediction
- **Loss weights**: Gender 0.3, Next-character 0.7
- **Embedding dimension**: 32
- **Hidden dimension**: 64
- **Performance**: AUC-ROC 0.94
