# Предсказание пола по имени и генерация имен с помощью RNN

Комплексный проект по глубокому обучению, исследующий различные архитектуры RNN для предсказания пола по именам и генерации новых имен.

## 🎯 Overview

В этом проекте исследуется применение различных архитектур рекуррентных нейронных сетей (RNN) для задач **классификации пола** и **генерации имен**. Реализованы и сравнены модели Vanilla RNN, GRU и LSTM с анализом их эффективности в понимании лингвистических паттернов в именах.

### Key Research Questions:
1. Какая архитектура RNN лучше всего подходит для классификации пола по именам?
2. Как различные модели RNN справляются с проблемой затухающих градиентов?
3. Насколько качественными получаются имена, сгенерированные каждой моделью?
4. Улучшает ли многозадачное обучение производительность модели?
5. Как различные функции потерь влияют на классификацию пола?

## ✨ Features

### Model Implementations
- **Vanilla RNN** - Кастомная реализация с нуля
- **GRU (Gated Recurrent Unit)** - Реализация на PyTorch
- **LSTM (Long Short-Term Memory)** - Реализация на PyTorch
- **Multi-task LSTM** - Комбинированная классификация пола и предсказание следующего символа

### Name Generation
- Предсказание следующего символа
- Сэмплинг с контролем температуры для генерации разнообразных имен
- Сравнение нескольких моделей

### Model Evaluation
- **Name Quality Score (NQS)** - Пользовательская метрика, оценивающая:
  - Плавность (внешний вид как у имени)
  - Правдоподобность (вероятность реального имени)
  - Разнообразие (вариативность сгенерированных имен)
  - Соответствие полу (согласованность предсказаний)

- **Perplexity** - Метрика "удивления" модели
- **AUC-ROC** - Производительность классификации
- **Анализ градиентов** - Исследование проблемы затухающих градиентов

## 📊 Dataset

**Источник**: Набор данных US Baby Names (очищенная версия)
- **Всего имен**: 6,782
- **Распределение по полу**: 
  - Девочки: 49.3%
  - Мальчики: 50.7%
- **Длина имени**: 2-11 символов
- **Предобработка**:
  - Приведение к нижнему регистру
  - Начальный/конечный токены (`!` и `#`)
  - Дополнение до максимальной длины
  - Токенизация на уровне символов (29 уникальных токенов)

## 🧠 Models Implemented

### 1. Vanilla RNN (Кастомная реализация)
- **Архитектура**: Кастомная RNN с ручным обратным распространением
- **Размер скрытого слоя**: 128
- **Назначение**: Базовая модель для сравнения
- **Производительность**: AUC-ROC 0.52

### 2. Модель GRU
- **Архитектура**: Реализация PyTorch GRU
- **Размер скрытого слоя**: 128
- **Размер эмбеддинга**: 64
- **Dropout**: 0.2
- **Производительность**: AUC-ROC 0.80

### 3. Модель LSTM
- **Архитектура**: Реализация PyTorch LSTM
- **Размер скрытого слоя**: 128
- **Размер эмбеддинга**: 64
- **Dropout**: 0.2
- **Производительность**: AUC-ROC 0.84

### 4. Многозадачная LSTM
- **Двойные цели**: Классификация пола + Предсказание следующего символа
- **Веса потерь**: Пол 0.3, Следующий символ 0.7
- **Размер эмбеддинга**: 32
- **Размер скрытого слоя**: 64
- **Производительность**: AUC-ROC 0.94

### Методы визуализации
- **t-SNE** и **PCA** для визуализации эмбеддингов
- **Графики норм градиентов** для анализа затухающих градиентов
- **Графики сходимости обучения**
- **ROC-кривые** для оценки классификации

### Детали обучения
- **Эпохи**: 20-100 в зависимости от модели
- **Скорость обучения**: 0.001-0.01
- **Ранняя остановка**: Терпение 10
- **Оптимизатор**: Adam
- **Валидационная выборка**: 10%
